True Gaze™ Parallax Correction

Standard eye trackers assume direct coaxial alignment between the camera and the eyes, causing significant gaze direction error when users shift off-axis. HCEP solves this by computing physical pupil offsets relative to the eye socket center in 3D space.

Estimated Gaze Angle Error 0.4°

True Gaze™ Geometry

Calculates dynamic gaze parallax by combining 3D face mesh PnP solvers with absolute pupil position mapping, yielding sub-degree precision even with basic webcams.

LLM & MCP Protocols

Cognitive Mode Analysis

    tools=[tool],

# Agent can now query state when user is looking at a specific window!

// Attach HcepGazeController to your 3D avatar rig
public class HcepGazeController : MonoBehaviour
{
    public Transform leftEyeBone;
    public Transform rightEyeBone;

    void Update()
    {
        // Reads smooth tracking values fetched from WebSocket stream /ws/stream
        Quaternion eyeRotation = Quaternion.Euler(eyePitch, eyeYaw, 0f);
        leftEyeBone.localRotation = eyeRotation;
        rightEyeBone.localRotation = eyeRotation;
    }
}
// Include HcepGazeController.h inside your Unreal C++ Character / Actor class
void AMyAvatarCharacter::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);

    // Retrieve current gaze target vectors smoothly interpolated by HCEP
    FVector GazeVector = GazeControllerComponent->TargetGazeDirection;
    FRotator HeadRot = GazeControllerComponent->TargetHeadRotation;

    // Rotate eye socket bones and mesh transforms directly in C++ or Blueprint AnimGraph
    LeftEyeMesh->SetRelativeRotation(GazeVector.Rotation());
    RightEyeMesh->SetRelativeRotation(GazeVector.Rotation());
}
